# ATAREFADO

**Matheus Maximiano Rabelo**

O objetivo desta aplicação é registrar, de forma simples, tarefas a serem realizadas até uma determinada data.
Essa é uma aplicação com fins acadêmicos voltada a disciplina de Tópicos Especiais: Aplicações Hibridas. Para tal, a aplicação foi desenvolvida usando apenas HTML, CSS e JavaScript.

## 1. Interfaces

### Tela principal

Nesta tela, são apresentadas as tarefas existentes e suas subtarefas, caso possua alguma.
 - Há um botão que direciona para a tela de cadastro das tarefas.

![127 0 0 1_5500_Pages_index html(iPhone 6_7_8)](https://user-images.githubusercontent.com/34040764/95666201-c450de00-0b2d-11eb-82fa-c4f5ac292ba1.png)

### Tela de cadastro

Nesta outra tela, o usuário conseguirá cadastrar novas tarefas e suas subtarefas caso existam.
 - Há um campo de texto para inserir a descrição da tarefa;
 - Há um campo de data para inserir a data alvo tarefa;
 - Toggle para informar se há ou não sub tarefas que, ao clicar, aparece campo para incluir descrição das subtarefas.


![127 0 0 1_5500_Pages_index html(iPhone 6_7_8) (1)](https://user-images.githubusercontent.com/34040764/95666542-02033600-0b31-11eb-9e75-73078d83f480.png)

![127 0 0 1_5500_Pages_index html(iPhone 6_7_8) (2)](https://user-images.githubusercontent.com/34040764/95666776-7f2faa80-0b33-11eb-8814-4fbfe2fdf138.png)


### Tela de edição

Nesta tela, o usuário conseguirá editar as tarefas e suas subtarefas caso existam.
 - A descrição da tarefa aparece, permitindo edição, assim como as subtarefas e a data;
 - Uma tarefa que possua subtarefas, poderá ter suas subtarefas removidas;
 - Uma tarefa sem subtarefas, poderá acrescentar subtarefas.

Nesta aplicação, não há armazenamento de dados do usuário!

## 3. Checklist de implementação

- A aplicação é original e não uma cópia da aplicação de um colega ou de uma aplicação já existente? **Sim**
- A aplicação tem pelo menos duas interfaces (telas ou páginas) independentes? **Sim**
- A aplicação armazena e usa de forma relevante dados complexos do usuário? **Não:** sem necessidade de manter dados do usuário.
- A aplicação possui um manifesto para instalação no dispositivo do usuário? **Sim**
- A aplicação possui um _service worker_ que permite o funcionamento off-line?**Sim**
- O código da minha aplicação possui comentários explicando cada operação?**Não** o nome dos métodos e das variáveis torna explicito o que cada um está fazendo.
- A aplicação está funcionando corretamente? **Sim**
- A aplicação está completa?**Sim**
